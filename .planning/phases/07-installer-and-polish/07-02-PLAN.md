    **`apps/api/src/routes/settings.ts`** — new file, register four routes:

    `GET /api/settings` — returns `{ diary_name, timezone }` for the authenticated user. Query `SELECT diary_name, timezone FROM users WHERE id = $1`.

    `PATCH /api/settings` — body `{ diary_name?: string, timezone?: string }`. Validate: diary_name max 80 chars, timezone must be a non-empty string (basic check, don't parse IANA). UPDATE users SET the provided fields. Return `{ ok: true }`.

    `GET /api/settings/thresholds` — returns array of `{ id, threshold_minutes, tone }` from notification_thresholds WHERE user_id = $1 ORDER BY threshold_minutes DESC.

    `PATCH /api/settings/thresholds` — body `{ thresholds: Array<{ threshold_minutes: number, tone: 'gentle'|'urgent'|'final' }> }`. DELETE existing rows for user, then INSERT new rows. Return `{ ok: true }`. Wrap in a transaction. Validate: threshold_minutes must be positive integer, tone must be one of the three valid values. Min 1 threshold, max 10.

    All routes use `preHandler: [requireAuth]`.

    **`apps/api/src/routes/entries.ts`** — add a new route to the existing file:

    `GET /api/entries/streak` — computes current streak. Query:
    ```sql
    SELECT DATE(created_at AT TIME ZONE u.timezone) AS entry_date
    FROM entries e
    JOIN users u ON u.id = e.user_id
    WHERE e.user_id = $1
    GROUP BY DATE(created_at AT TIME ZONE u.timezone)
    ORDER BY entry_date DESC
    ```
    Walk the rows: start from today (in user's timezone), count consecutive days with at least one entry. Stop at first gap. Return `{ streak: N, last_entry_date: "YYYY-MM-DD" | null }`. If no entries, return `{ streak: 0, last_entry_date: null }`.

    **`apps/api/src/server.ts`** — add `import settingsRoutes from "./routes/settings.js"` and `fastify.register(settingsRoutes)` after the existing route registrations.

    Validation Architecture: `GET /api/settings/thresholds` with a valid session returns 200 with array. `GET /api/entries/streak` returns `{ streak: 0 }` for new user. Verified by curl in phase verification.
