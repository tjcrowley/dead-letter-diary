  <name>Task 1: Vitest web config, crypto module, and crypto tests</name>
  <files>apps/web/package.json, apps/web/vitest.config.ts, apps/web/lib/crypto.ts, apps/web/lib/__tests__/crypto.test.ts</files>
  <behavior>
    - generateDmk() returns an extractable CryptoKey with AES-GCM 256-bit
    - wrapDmk(dmk, deviceShard, serverShard, hkdfSalt) returns { wrappedDmk: ArrayBuffer, wrapIv: Uint8Array }
    - unwrapDmk(wrappedDmk, wrapIv, deviceShard, serverShard, hkdfSalt) returns a NON-extractable CryptoKey
    - Round-trip: generate -> wrap -> unwrap produces a key that can encrypt/decrypt identically
    - unwrapDmk result has extractable === false (verify via crypto.subtle.exportKey throwing)
    - encryptEntry(dmk, plaintext, entryId, userId, wordCount) returns { ciphertext, iv, aad }
    - decryptEntry(dmk, ciphertext, iv, aad) returns original plaintext string
    - Two calls to encryptEntry with same inputs produce different IVs
    - decryptEntry with wrong AAD throws OperationError
    - deriveShardFromPassphrase(passphrase, salt) returns Uint8Array of length 32
    - xorShards(a, b) returns XOR'd Uint8Array; throws if lengths differ
  </behavior>
  <action>
    1. Install vitest and happy-dom in web workspace: `cd apps/web && npm install -D vitest happy-dom`
    2. Create `apps/web/vitest.config.ts` with `environment: "happy-dom"` (happy-dom supports Web Crypto; jsdom does NOT)
    3. Write failing tests in `apps/web/lib/__tests__/crypto.test.ts` covering all behaviors above
    4. Implement `apps/web/lib/crypto.ts` with these exports:
       - `generateDmk()`: Uses `crypto.subtle.generateKey("AES-GCM", 256, true, ["encrypt","decrypt"])` — extractable=true for initial wrap
       - `wrapDmk(dmk, deviceShard, serverShard, hkdfSalt)`: XOR shards -> importKey HKDF (non-extractable) -> deriveKey AES-GCM -> wrapKey with fresh 12-byte IV. Info string: `"dead-letter-diary-dmk-wrap"`
       - `unwrapDmk(wrappedDmk, wrapIv, deviceShard, serverShard, hkdfSalt)`: Same HKDF derivation -> unwrapKey with `extractable: false` (CRYPT-08)
       - `encryptEntry(dmk, plaintext, entryId, userId, wordCount)`: Fresh 12-byte IV (CRYPT-06), AAD = JSON.stringify({entryId, userId, wordCount}) encoded as UTF-8 (CRYPT-07), AES-GCM encrypt
       - `decryptEntry(dmk, ciphertext, iv, aad)`: AES-GCM decrypt with provided AAD
       - `deriveShardFromPassphrase(passphrase, salt)`: PBKDF2-SHA256 with 600,000 iterations, deriveBits(256) -> Uint8Array(32) (CRYPT-03)
       - `xorShards(a, b)`: Byte-wise XOR, throws if lengths differ
       - Helper: `uint8ToBase64url(bytes)`, `base64urlToUint8(b64)` for transport encoding
    5. Run tests and ensure all pass

    IMPORTANT CONSTRAINTS:
    - Web Crypto API ONLY — no third-party crypto libraries (project constraint)
    - HKDF importKey MUST set extractable=false (Web Crypto spec requirement — will throw InvalidAccessError otherwise)
    - DMK generated as extractable=true (for wrapKey), then unwrapped as extractable=false (for session use)
    - AAD must be deterministic (JSON.stringify with fixed key order) so encrypt/decrypt AAD matches exactly
  </action>
