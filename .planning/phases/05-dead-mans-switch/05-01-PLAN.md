  <behavior>
    - computeDeadlineUTC(24, "America/New_York") returns a UTC Date that reflects the correct wall-clock time including DST (use Luxon DateTime.now().setZone(tz).plus({ hours }).toUTC())
    - checkDeadlines(pool, log): for each user with state='active' and deadline_at <= now, calls initiateWipe which: acquires FOR UPDATE lock → inserts wipe_log row → sets state='pending_wipe'
    - checkDeadlines: for each user with state='pending_wipe' and wipe_log.initiated_at <= now - 60s, calls confirmWipe which: acquires FOR UPDATE lock → sets wipe_log.shard_deleted=true → sets state='wiped'
    - confirmWipe runs DELETE FROM server_shards only AFTER wipe_log.shard_deleted is set (crash safety: DMS-08)
    - initiateWipe is a no-op if state is already 'pending_wipe' or 'wiped' (idempotent)
    - checkDeadlines also promotes Akrasia pending changes: UPDATE deadline_state SET window_hours=pending_window_hours, word_minimum=pending_word_minimum WHERE pending_effective_at IS NOT NULL AND pending_effective_at <= now()
    - checkDeadlines checks notification thresholds: for each user, for each threshold in notification_thresholds, fire if now is within the 60s window after crossing the threshold (thresholdMs >= prevPollMs && thresholdMs <= nowMs). Threshold firing stores a row in notifications.last_sent_at to prevent duplicates (not implemented in this plan — stub the send call as a no-op logger for now; Plan 02 wires the real sender)
  </behavior>

    - `computeDeadlineUTC(windowHours: number, ianaTimezone: string): Date` — uses Luxon DateTime.now().setZone(ianaTimezone).plus({ hours: windowHours }).toUTC().toJSDate()
    - `initiateWipe(client: PoolClient, userId: string): Promise<void>` — BEGIN, SELECT deadline_state WHERE user_id=$1 FOR UPDATE, check state==='active', INSERT INTO wipe_log (user_id, reason, initiated_at) VALUES ($1, 'deadline', now()), UPDATE deadline_state SET state='pending_wipe' updated_at=now(), COMMIT.     - `confirmWipe(client: PoolClient, userId: string): Promise<void>` — BEGIN, SELECT deadline_state WHERE user_id=$1 FOR UPDATE, check state==='pending_wipe', check wipe_log WHERE user_id=$1 AND confirmed_at IS NULL ORDER BY initiated_at DESC LIMIT 1 — verify initiated_at is <= now()-60s, UPDATE wipe_log SET shard_deleted=true, confirmed_at=now(), DELETE FROM server_shards WHERE user_id=$1, UPDATE deadline_state SET state='wiped' updated_at=now(), COMMIT.
