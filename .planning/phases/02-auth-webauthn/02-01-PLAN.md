    - Test unlock: rejects invalid passphrase (401)

  <action>
    1. Create `apps/api/src/routes/auth.ts`:
       - `POST /api/auth/register`: Single-user app check (`SELECT COUNT(*) FROM users`; reject 409 if > 0). Validate passphrase minimum 12 chars. Hash with `argon2.hash(passphrase, { type: argon2.argon2id, memoryCost: 65536, timeCost: 3, parallelism: 4 })`. Generate `hkdf_salt` as `crypto.randomBytes(32)`. INSERT into users. Create session: generate `crypto.randomBytes(32)` token, SHA-256 hash it, INSERT into sessions with 7-day expiry. Sign JWT with `{ sub: userId, sid: sessionId }` and 7d expiresIn. Set httpOnly secure sameSite=strict cookie named `session`. Return 201 with `{ id: userId, prfCapable: false }`.
       - `POST /api/auth/unlock`: Get single user (`SELECT id, passphrase_hash FROM users LIMIT 1`; 404 if none). Verify with `argon2.verify(hash, passphrase)`. If invalid: 401. If valid: create session same as register. Return 200 with `{ id: userId }` and `hkdf_salt` as base64url (for Phase 3 client key derivation).
       - `GET /api/auth/me`: Protected by requireAuth preHandler. Returns `{ id: request.userId }`. This validates the session cookie works.
       - `DELETE /api/auth/session`: Protected by requireAuth. Deletes session from sessions table by sid from JWT. Clears cookie.

    2. Create test files:
       - `apps/api/src/routes/__tests__/auth.test.ts`: Tests for register (happy path, duplicate, short passphrase) and unlock (happy path, wrong passphrase, no account). Use `fastify.inject()` to test routes. Mock pg pool to control DB responses. Verify argon2id hash format in register. Verify cookie is set with httpOnly/secure flags.
       - `apps/api/src/routes/__tests__/session.test.ts`: Tests for /api/auth/me (valid session, no cookie, expired session). Tests for DELETE /api/auth/session (logout clears cookie + session row).
