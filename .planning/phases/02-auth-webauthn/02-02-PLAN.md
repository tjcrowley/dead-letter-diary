  <action>
    1. Create `apps/api/src/routes/webauthn.ts` with four endpoints, all under prefix `/api/webauthn`:

    **POST /api/webauthn/register-options** (requireAuth — user must be logged in via passphrase first):
    - Get user_id from request.userId.
    - Get existing credentials: `SELECT id, transports FROM webauthn_credentials WHERE user_id = $1`.
    - Call `generateRegistrationOptions({ rpName: 'Dead Letter Diary', rpID: process.env.WEBAUTHN_RP_ID || 'localhost', userName: userId, attestationType: 'none', excludeCredentials: existingCreds, authenticatorSelection: { residentKey: 'required', userVerification: 'required', authenticatorAttachment: 'platform' } })`.
    - Store challenge in Redis with key `webauthn:challenge:${userId}` and 60-second TTL.
    - Return options JSON to client. Client will add PRF extension on its side.

    **POST /api/webauthn/register-verify** (requireAuth):
    - Get stored challenge from Redis: `GET webauthn:challenge:${userId}`. Delete immediately after read (single-use).
    - If no challenge found: 400 `{ error: 'Challenge expired or already used' }`.
    - Call `verifyRegistrationResponse({ response: request.body.attestation, expectedChallenge: storedChallenge, expectedOrigin: \`https://${process.env.WEBAUTHN_RP_ID || 'localhost'}\`, expectedRPID: process.env.WEBAUTHN_RP_ID || 'localhost' })`.
    - If not verified: 400 `{ error: 'Registration verification failed' }`.
    - Extract `registrationInfo`: `credential.id`, `credential.publicKey`, `credential.counter`.
    - Extract `prf_capable` from `request.body.prfEnabled` (boolean sent by client after checking `clientExtensionResults.prf?.enabled`).
    - INSERT into webauthn_credentials: `(id, user_id, public_key, counter, transports, prf_capable)`.
    - Return 201 `{ credentialId, prfCapable }`.

    **POST /api/webauthn/auth-options** (public — no auth required):
    - Get the single user: `SELECT id FROM users LIMIT 1`. 404 if none.
    - Get credentials: `SELECT id, transports FROM webauthn_credentials WHERE user_id = $1`.
    - 404 if no credentials registered.
    - Call `generateAuthenticationOptions({ rpID: process.env.WEBAUTHN_RP_ID || 'localhost', allowCredentials: creds.map(c => ({ id: c.id, transports: c.transports })), userVerification: 'required' })`.
    - Store challenge in Redis: `webauthn:challenge:auth:${userId}` with 60s TTL.
    - Return options JSON. Also return the user's `hkdf_salt` as base64url so the client can use it as PRF eval salt.

    **POST /api/webauthn/auth-verify** (public):
    - Get the single user: `SELECT id, hkdf_salt FROM users LIMIT 1`.
    - Get stored challenge from Redis. Delete immediately.
    - Find credential: `SELECT id, public_key, counter, transports FROM webauthn_credentials WHERE id = $1 AND user_id = $2` using `request.body.response.id`.
    - Call `verifyAuthenticationResponse({ response: request.body.response, expectedChallenge: storedChallenge, expectedOrigin: \`https://${process.env.WEBAUTHN_RP_ID || 'localhost'}\`, expectedRPID: process.env.WEBAUTHN_RP_ID || 'localhost', credential: { id: cred.id, publicKey: cred.public_key, counter: Number(cred.counter), transports: cred.transports } })`.
    - **AUTH-07**: If `!verification.authenticationInfo.userVerified`: return 403 `{ error: 'Biometric confirmation required (UV flag not set)' }`.
    - Update counter: `UPDATE webauthn_credentials SET counter = $1 WHERE id = $2`.
    - Create session (same pattern as auth.ts register/unlock): random token, SHA-256 hash, INSERT sessions, sign JWT, set cookie.
    - Return 200 `{ id: userId }`.

    2. Create `apps/api/src/routes/__tests__/webauthn.test.ts`:
    - For testing, mock `@simplewebauthn/server` functions (generateRegistrationOptions, verifyRegistrationResponse, etc.) using `vi.mock()`.
    - Mock redis with an in-memory Map for challenge storage.
    - Test each endpoint per the behavior list above.
    - For UV test: mock `verifyAuthenticationResponse` to return `{ verified: true, authenticationInfo: { userVerified: false, newCounter: 1 } }` and assert 403 response.

    3. Register webauthn routes in `apps/api/src/server.ts`:
    - `import webauthnRoutes from './routes/webauthn.js'`
    - `fastify.register(webauthnRoutes)`
