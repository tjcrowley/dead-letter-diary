<!-- WebAuthn browser patterns from RESEARCH.md -->
```typescript
// Registration with PRF:
const attResp = await startRegistration({
  optionsJSON: { ...options, extensions: { ...options.extensions, prf: {} } },
});
const prfEnabled = attResp.clientExtensionResults?.prf?.enabled;

// Authentication with PRF eval:
const authResp = await startAuthentication({
  optionsJSON: {
    ...options,
    extensions: { prf: { eval: { first: prfSaltBytes } } },
  },
});
const prfResult = authResp.clientExtensionResults?.prf?.results?.first;
// prfResult stays client-side only — NEVER sent to server
```

    2. Create `apps/web/lib/api.ts`:
       - Export an `api` object with `get`, `post`, `delete` methods.
       - Base URL from `process.env.NEXT_PUBLIC_API_URL || ''` (empty = same origin, Caddy proxies /api to Fastify).
       - All methods: `credentials: 'include'` (sends cookies cross-origin if needed), `Content-Type: application/json`.
       - Response handler: parse JSON, throw on non-ok status with error message from body.

    3. Create `apps/web/lib/webauthn.ts`:
       - `registerPasskey()`: calls `api.post('/api/webauthn/register-options')` to get options. Adds PRF extension `{ prf: {} }` to options.extensions. Calls `startRegistration({ optionsJSON: modifiedOptions })`. Extracts `prfEnabled = attResp.clientExtensionResults?.prf?.enabled ?? false`. Calls `api.post('/api/webauthn/register-verify', { attestation: attResp, prfEnabled })`. Returns `{ credentialId, prfCapable: prfEnabled }`.
       - `authenticatePasskey()`: calls `api.post('/api/webauthn/auth-options')` to get `{ options, hkdfSalt }`. Converts hkdfSalt from base64url to ArrayBuffer using `base64URLStringToBuffer` from @simplewebauthn/browser. Adds PRF eval extension: `{ prf: { eval: { first: saltBuffer } } }`. Calls `startAuthentication({ optionsJSON: modifiedOptions })`. Extracts `prfResult = authResp.clientExtensionResults?.prf?.results?.first` (ArrayBuffer, stays client-side). Calls `api.post('/api/webauthn/auth-verify', { response: authResp })`. Returns `{ userId, prfResult }` — prfResult is kept in memory for Phase 3 key derivation, NEVER sent to server.

    4. Create `apps/web/app/setup/page.tsx` ('use client'):
       - Two-step flow: Step 1 = Create account (passphrase input + confirm), Step 2 = Enroll passkey.
       - Step 1: Passphrase input (type="password", min 12 chars), confirm passphrase field, "Create Account" button. Calls `api.post('/api/auth/register', { passphrase })`. On success, moves to Step 2.
       - Step 2: "Register Passkey" button. Calls `registerPasskey()`. Shows result: "Passkey registered (PRF: supported/not supported)". If PRF not supported, show info: "Your device doesn't support PRF. You'll use passphrase for encryption key derivation."
       - After both steps: redirect to `/` (diary will be implemented in Phase 3).
       - Minimal styling with Tailwind-like inline styles or CSS modules (match the dark minimal theme from Phase 1 — "Dead Letter Diary" branding). No heavy styling framework needed.
       - Error handling: show error messages inline below inputs.

    5. Create `apps/web/app/unlock/page.tsx` ('use client'):
       - **AUTH-06**: Both unlock methods visible simultaneously, not behind tabs.
       - Layout: Two columns (or stacked on mobile).
         - Left/Top: "Biometric Unlock" section. Large "Unlock with Face ID / Touch ID" button that calls `authenticatePasskey()`. Shows status/errors inline.
         - Right/Bottom: "Passphrase" section. Passphrase input + "Unlock" button that calls `api.post('/api/auth/unlock', { passphrase })`.
       - **AUTH-05 (PIN)**: Below both sections, a "Quick PIN" section.
         - If no PIN set: "Set PIN" link that shows a 4-6 digit input + confirm. Stores PIN in `sessionStorage` (cleared on tab close). PIN is compared client-side only — no server call.
         - If PIN is set: 4-6 digit input. On correct PIN: check if session cookie is still valid via `api.get('/api/auth/me')`. If valid, redirect to `/`. If expired, show "Session expired, use passphrase or biometric" and clear PIN from sessionStorage.
         - PIN NEVER leaves the browser. No server endpoint. No PIN hash in DB.
       - After successful unlock (any method): redirect to `/`.
       - Error states: "Invalid passphrase", "Biometric failed", "No account found — go to /setup".
