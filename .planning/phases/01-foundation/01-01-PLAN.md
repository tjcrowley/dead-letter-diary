Create the comprehensive schema covering ALL tables for all 7 phases (locked decision: scaffold all tables in Phase 1). Every table must have `user_id` (multi-user-ready, locked decision).

**001.do.create-schema.sql:** Wrap in BEGIN/COMMIT transaction. Create tables in dependency order:

1. `users` — id (UUID PK default gen_random_uuid()), passphrase_hash TEXT NOT NULL, hkdf_salt BYTEA NOT NULL (32-byte, CRYPT-09), epitaph TEXT (WIPE-04), timezone TEXT NOT NULL DEFAULT 'UTC', created_at/updated_at TIMESTAMPTZ NOT NULL DEFAULT now()

2. `sessions` — id UUID PK, user_id UUID NOT NULL FK users(id) ON DELETE CASCADE, token_hash TEXT NOT NULL UNIQUE, expires_at TIMESTAMPTZ NOT NULL, created_at

3. `webauthn_credentials` — id TEXT PK (credential ID from authenticator), user_id FK, public_key BYTEA NOT NULL, counter BIGINT NOT NULL DEFAULT 0, transports TEXT[], device_name TEXT, created_at

4. `key_wraps` — id UUID PK, user_id FK, credential_id TEXT FK webauthn_credentials(id) ON DELETE CASCADE, wrapped_dmk BYTEA NOT NULL, wrap_iv BYTEA NOT NULL (12-byte), wrap_type TEXT NOT NULL ('webauthn_prf' | 'passphrase'), created_at

5. `server_shards` — id UUID PK, user_id FK, shard BYTEA NOT NULL, created_at

6. `entries` — id UUID PK, user_id FK, ciphertext BYTEA NOT NULL, iv BYTEA NOT NULL (12-byte), aad BYTEA, word_count INT NOT NULL, created_at/updated_at

7. `deadline_state` — id UUID PK, user_id UNIQUE FK, window_hours INT NOT NULL DEFAULT 24, word_minimum INT NOT NULL DEFAULT 50, deadline_at TIMESTAMPTZ, state TEXT NOT NULL DEFAULT 'active' (active|pending_wipe|wiped), grace_used_at TIMESTAMPTZ, grace_budget INT NOT NULL DEFAULT 1, pending_window_hours INT, pending_word_minimum INT, pending_effective_at TIMESTAMPTZ, created_at/updated_at

8. `wipe_log` — id UUID PK, user_id FK, reason TEXT NOT NULL ('deadline'|'panic'), initiated_at TIMESTAMPTZ NOT NULL DEFAULT now(), confirmed_at TIMESTAMPTZ, shard_deleted BOOLEAN NOT NULL DEFAULT false

9. `notifications` — id UUID PK, user_id FK, subscription JSONB NOT NULL, created_at/updated_at

10. `notification_thresholds` — id UUID PK, user_id FK, threshold_minutes INT NOT NULL, tone TEXT NOT NULL ('gentle'|'urgent'|'final'), created_at

Indexes (after all CREATE TABLE statements):
- idx_sessions_user_id ON sessions(user_id)
- idx_sessions_expires_at ON sessions(expires_at)
- idx_entries_user_id_created ON entries(user_id, created_at DESC)
- idx_deadline_state_deadline ON deadline_state(deadline_at) WHERE state = 'active'
- idx_wipe_log_user_id ON wipe_log(user_id)
- idx_webauthn_credentials_user ON webauthn_credentials(user_id)
- idx_key_wraps_user ON key_wraps(user_id)
- idx_notifications_user ON notifications(user_id)

**001.undo.create-schema.sql:** DROP all tables in reverse dependency order, wrapped in BEGIN/COMMIT. Drop indexes implicitly via DROP TABLE CASCADE.
