  <action>
Replace the placeholder server.ts with a real Fastify application.

**apps/api/src/plugins/db.ts:** Create a Fastify plugin that:
- Creates a `pg.Pool` using `DATABASE_URL` from environment
- Decorates `fastify.pg` with the pool instance (use `fastify.decorate('pg', pool)`)
- Adds an `onClose` hook to end the pool on shutdown
- Add TypeScript declaration merging to extend FastifyInstance with `pg: Pool`

**apps/api/src/boot/migrate.ts:** Create the migration runner following RESEARCH.md Pattern 1:
- Export `async function runMigrations(connectionString: string)`
- Create a `pg.Client`, connect, create Postgrator instance
- Configure: `migrationPattern` pointing to `../../migrations/*` (use `path.join(__dirname, ...)`), driver 'pg', schemaTable 'schema_migrations', execQuery using client.query
- Run `postgrator.migrate()`, log count of applied migrations
- Close client when done
- Handle errors: log and re-throw (don't swallow migration failures)

**apps/api/src/routes/health.ts:** Create the health check route:
- Export an async function that takes FastifyInstance
- Register `GET /api/health`
- Try `await fastify.pg.query('SELECT 1')` -- if it fails, return 503 with `{ status: 'unhealthy', database: 'unreachable' }`
- On success, return `{ status: 'healthy', timestamp: new Date().toISOString() }`

**apps/api/src/server.ts:** Create the Fastify entry point:
- Import dotenv and call `config()` at the top
- Create Fastify instance with `logger: true`
- Register the db plugin
- Register health routes
- Call `runMigrations(process.env.DATABASE_URL!)` before `fastify.listen()`
- Listen on `{ port: Number(process.env.PORT) || 3001, host: '0.0.0.0' }`
- Add graceful shutdown: handle SIGINT and SIGTERM, call `fastify.close()`
- Wrap everything in an async `start()` function with try/catch that logs errors and exits with code 1

Use CommonJS-compatible imports (require-style or esModuleInterop) consistent with the tsconfig from Plan 01. The Fastify 5 API is TypeScript-first but the project uses NodeNext module resolution -- follow that.
